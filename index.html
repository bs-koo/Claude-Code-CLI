02

Claude Code 설치 (Windows)

세 가지 설치 방법 중 환경에 맞게 선택

자동 업데이트 지원. Node.js 설치 불필요.

PowerShell

powershell 
irm https://claude.ai/install.ps1 | iex

CMD

batch 
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd

Windows 패키지 관리자를 통한 설치. 수동 업데이트 필요.

powershell 
winget install Anthropic.ClaudeCode
⚠
수동 업데이트 필요: winget upgrade Anthropic.ClaudeCode
✕
Deprecated: npm 설치 방식은 더 이상 권장되지 않습니다. Node.js 18+ 필요.
bash 
npm install -g @anthropic-ai/claude-code

설치 확인

bash 
claude --version
claude doctor

첫 실행

bash 
cd your-project
claude
ℹ
첫 실행 시 브라우저를 통한 로그인 프롬프트가 표시됩니다. Claude Pro/Max/Team/Enterprise 계정 또는 Anthropic Console API 키가 필요합니다.

06

oh-my-claudecode 플러그인

멀티 에이전트 오케스트레이션 플러그인

⚡
19개 특수 에이전트 · 28개 스킬 · 5개 실행 모드를 제공하는 멀티 에이전트 오케스트레이션 플러그인입니다. 복잡한 작업을 자율적으로 분담하고 병렬 실행합니다.

설치 방법

bash 
/plugin marketplace add https://github.com/Yeachan-Heo/oh-my-claudecode
/plugin install oh-my-claudecode
ℹ
리포지토리명은 oh-my-claudecode이지만 npm 패키지명은 oh-my-claude-sisyphus입니다.
bash 
npm i -g oh-my-claude-sisyphus@latest

설치 후 초기 설정

slash command 
/oh-my-claudecode:omc-setup

팀 기능 활성화

경로: ~/.claude/settings.json

json 
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}

주요 실행 모드 (매직 키워드)

autopilot
아이디어에서 동작하는 코드까지 자율 실행
ralph
아키텍트 검증 포함 지속 루프
ulw (ultrawork)
최대 병렬 실행 모드
plan
인터뷰 기반 전략 계획 수립
deep-interview
소크라테스식 요구사항 명확화
team
조율된 에이전트 팀 동시 실행

19개 에이전트

빌드 & 분석

Explore Analyst Planner Architect Debugger Executor Code-Simplifier

리뷰 & 품질

Security-Reviewer Code-Reviewer Critic

도메인 전문가

Document-Specialist Test-Engineer Designer Writer QA-Tester Scientist Git-Master Tracer

HUD (Status Line) 설정

현재 상태를 실시간으로 표시하는 상태 표시줄

7-1. Claude Code 기본 Status Line

slash command 
/statusline show model name and context percentage with a progress bar

수동 설정 — Linux/macOS

경로: ~/.claude/settings.json

json 
{
  "statusLine": {
    "type": "command",
    "command": "~/.claude/statusline.sh"
  }
}

수동 설정 — Windows PowerShell

json 
{
  "statusLine": {
    "type": "command",
    "command": "powershell -NoProfile -File C:/Users/username/.claude/statusline.ps1"
  }
}

사용 가능한 데이터 필드

필드설명
model.display_name현재 사용 중인 모델명
context_window.used_percentage컨텍스트 윈도우 사용률 (%)
cost.total_cost_usd세션 누적 비용 (USD)
cost.total_duration_ms세션 경과 시간
rate_limits.five_hour.used_percentage5시간 Rate Limit 사용률
rate_limits.seven_day.used_percentage7일 Rate Limit 사용률
workspace.current_dir현재 작업 디렉토리

7-2. oh-my-claudecode HUD

slash command 
/oh-my-claudecode:hud setup

프리셋 전환

slash command 
/oh-my-claudecode:hud minimal    # 최소 표시
/oh-my-claudecode:hud focused    # 기본값
/oh-my-claudecode:hud full       # 전체 표시

프리셋 비교

minimal
[OMC] ralph · ultrawork · todos:2/5
focused
[OMC] repo:name branch:main ralph:3/10 ctx:67% agents:2 bg:3/5 todos:2/5
full
[OMC] repo:name branch:main ralph:3/10 ctx:67% agents:2 bg:3/5 todos:2/5 ├─ Explore: running ctx:32% └─ Executor: running ctx:45%

수동 설정 (settings.json)

json 
{
  "omcHud": {
    "preset": "focused",
    "elements": {
      "cwd": true,
      "gitRepo": true,
      "gitBranch": true,
      "omcLabel": true,
      "contextBar": true,
      "agents": true,
      "todos": true,
      "ralph": true,
      "autopilot": true,
      "showTokens": true,
      "backgroundTasks": true,
      "thinking": true,
      "permissionStatus": true
    }
  }
}

표시 요소 설명

요소설명
[OMC]OMC 모드 식별자
repo:name리포지토리 이름
branch:name현재 git 브랜치
ralph:3/10Ralph 루프 진행도
ctx:67%컨텍스트 사용률
agents:2실행 중인 서브에이전트 수
bg:3/5백그라운드 작업 슬롯 (사용/전체)
todos:2/5TODO 완료 비율
◉
색상 임계값: 초록 — 정상 / 노랑 — 70% 이상 / 빨강 — 85% 이상
08

RTK (Rust Token Killer)

토큰 사용량을 60–90% 절감하는 CLI 래퍼 도구

60–90%
평균 토큰 절감율 — 30분 세션 기준 ~118,000 → ~23,900 토큰
✂
Smart Filtering
주석, 공백, 보일러플레이트 자동 제거
⊞
Grouping
유사 항목 집계 및 그룹화
⋯
Truncation
관련 컨텍스트만 유지하고 나머지 생략
◑
Deduplication
반복 로그를 카운트로 축약

30분 세션 토큰 절감 비교

명령어원본RTK절감
ls/tree (10회) 2,000 400 -80%
cat/read (20회) 40,000 12,000 -70%
git status (10회) 3,000 600 -80%
git diff (5회) 10,000 2,500 -75%
cargo test / npm test (5회) 25,000 2,500 -90%
합계 ~118,000 ~23,900 -80%

설치 (Windows)

ℹ
GitHub Releases 페이지에서 rtk-x86_64-pc-windows-msvc.zip을 다운로드한 후 압축 해제하고 PATH에 추가하세요.
↗ GitHub Releases 열기
⚠
주의: cargo install rtk는 동명의 다른 패키지입니다. 반드시 --git 옵션을 사용하세요.
bash 
cargo install --git https://github.com/rtk-ai/rtk
bash 
curl -fsSL https://raw.githubusercontent.com/rtk-ai/rtk/refs/heads/master/install.sh | sh

설치 확인

bash 
rtk --version
rtk gain

Claude Code 연동

bash 
rtk init -g
✓
자동 생성: Hook 파일 · RTK.md · CLAUDE.md 참조 추가

주요 명령어 예시

bash 
rtk git status          # 압축된 git 상태
rtk git diff            # 압축된 diff (-75%)
rtk git log -n 10       # 한 줄 커밋 로그
rtk ls .                # 토큰 최적화 디렉토리 트리
rtk test cargo test     # 실패 테스트만 표시 (-90%)

절감 통계 확인

bash 
rtk gain                # 절감 통계 요약
rtk gain --graph        # 최근 30일 ASCII 그래프
rtk discover            # RTK 미사용 명령 탐지
09

터미널 Alias 설정 및 실행 모드

빠른 실행을 위한 단축어와 권한 관리

Alias 설정

경로: ~/.bashrc 에 추가

bash 
# ~/.bashrc에 추가
alias cc='claude'
alias ccd='claude --dangerously-skip-permissions'
alias ccr='claude --resume --dangerously-skip-permissions'

사용법

bash 
cd your-project       # 작업할 프로젝트 폴더로 이동
cc                    # 기본 모드
ccd                   # 바이패스 모드
ccr                   # 이전 세션 재개 + 바이패스

기본 모드 vs 바이패스 모드

항목기본 모드 (cc)바이패스 모드 (ccd)
파일 읽기/쓰기 매번 사용자 확인 자동 허용
셸 명령 실행 매번 사용자 확인 자동 허용
작업 속도 느림 (매번 승인 필요) 빠름 (중단 없이 진행)
안전성 최대 권한 설정으로 보호
🛡
바이패스 모드 안전성: --dangerously-skip-permissions는 이름과 달리 아래의 세 단계 권한 설정(allow/deny/ask)이 적용된 상태에서 동작합니다. deny 목록의 명령은 바이패스 모드에서도 완전히 차단됩니다. 프로덕션 서버나 민감한 환경에서는 기본 모드를 사용하세요.

권한 3단계 구조

레벨동작예시
allow 자동 허용 git status, ls, grep, 파일 읽기/쓰기, npm 등
deny 완전 차단 rm -rf /, git push --force, DROP DATABASE, shutdown
ask 사용자 확인 rm, sudo, chmod, .env 파일 읽기

settings.json 전체 예시

json 
{
  "permissions": {
    "allow": [
      "Edit", "Read", "Write", "Glob", "Grep",
      "Bash(git status:*)", "Bash(git add:*)", "Bash(git commit:*)",
      "Bash(git push:*)", "Bash(git pull:*)", "Bash(git diff:*)",
      "Bash(ls:*)", "Bash(cat:*)", "Bash(find:*)", "Bash(grep:*)",
      "Bash(npm*)", "Bash(node*)", "Bash(python*)",
      "Bash(mkdir:*)", "Bash(cp:*)", "Bash(mv:*)"
    ],
    "deny": [
      "Bash(rm -rf /:*)", "Bash(rm -rf ~:*)",
      "Bash(git push --force:*)", "Bash(git reset --hard:*)",
      "Bash(DROP DATABASE:*)", "Bash(DROP TABLE:*)",
      "Bash(shutdown:*)", "Bash(reboot:*)",
      "Bash(format:*)", "Bash(dd:*)"
    ],
    "ask": [
      "Bash(rm:*)", "Bash(sudo:*)", "Bash(chmod:*)",
      "Read(./.env)", "Read(./.env.*)"
    ]
  }
}

핵심 보안 보호

⛔
시스템 파괴 방지
rm -rf /, format, dd, shutdown 완전 차단
⚠
Git 안전
git push --force, git reset --hard 차단
🗄
DB 보호
DROP DATABASE, DROP TABLE, TRUNCATE 차단
?
삭제 시 확인
rm은 항상 사용자 확인 요청
🔑
환경변수 보호
.env 파일 읽기 시 사용자 확인

유용한 사용 패턴

실전에서 바로 쓰는 Claude Code 활용 예시

파일 분석 및 버그 탐지

bash 
claude "이 파일의 버그를 찾아줘 @./src/api.ts"

로그 파이프라인 분석

bash 
tail -200 app.log | claude -p "이 로그에서 이상한 점을 찾아줘"

보안 감사 (Git diff 활용)

bash 
git diff main --name-only | claude -p "변경된 파일들의 보안 이슈를 점검해줘"

테스트 작성 → 실행 → 수정 자동화

bash 
claude "auth 모듈 테스트를 작성하고 실행한 뒤 실패하면 수정해줘"
